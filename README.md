# Topik-Enkripsi-dan-Deskripsi
Topik ini buat pembelajaran soal keamanan siber dalam serangan Ransomware 

Repository ini membahas konsep fundamental enkripsi dan deskripsi, dua mekanisme utama yang digunakan untuk melindungi data digital dari akses tidak sah. Materi ini disusun untuk membantu pemula memahami cara kerja kriptografi modern dan bagaimana enkripsi menjaga kerahasiaan, integritas, serta keamanan informasi.

1. Pengertian Enkripsi
Enkripsi adalah proses mengubah data asli (plaintext) menjadi bentuk yang tidak dapat dibaca (ciphertext) menggunakan sebuah algoritma dan kunci. Tujuannya mencegah pihak yang tidak memiliki kunci membaca data tersebut.


2. Pengertian Deskripsi
Deskripsi adalah proses mengembalikan ciphertext menjadi plaintext menggunakan kunci yang sesuai. Hanya pihak yang memiliki kunci yang benar dapat mendekripsi data.


3. Jenis‑Jenis Sistem Kunci

Kriptografi Simetris:
Menggunakan satu kunci untuk enkripsi dan deskripsi. Contoh: AES.

Kriptografi Asimetris:
Menggunakan dua kunci, yaitu public key dan private key. Public key untuk mengenkripsi, private key untuk mendekripsi. Contoh: RSA, ECC.



4. Algoritma Umum yang Dipakai:

AES (Advanced Encryption Standard) → enkripsi simetris

RSA (Rivest‑Shamir‑Adleman) → enkripsi asimetris

ECC (Elliptic Curve Cryptography) → versi modern RSA yang lebih efisien



5. Fungsi Enkripsi Dalam Dunia Nyata:

Pengamanan komunikasi (HTTPS, VPN, SSH)

Penyimpanan file yang aman

Sistem login dan autentikasi

Protokol keamanan aplikasi


6. Studi Kasus Edukatif:
Contoh sederhana bagaimana ransomware memanfaatkan enkripsi, serta bagaimana analis keamanan mempelajari dan memahami teknik tersebut secara legal di lingkungan lab.
